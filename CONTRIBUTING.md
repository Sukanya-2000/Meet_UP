# Contributing

- Branch from the protected default branch and keep changes scoped.
- Add regression tests for every behavior change and migrations for additive schema changes.
- Preserve public APIs or document/version intentional breaks.
- Run lint, web build, backend tests and Flutter analysis/tests.
- Security and privacy changes require threat-model notes. Payment changes require webhook/idempotency tests.
- Do not merge with unresolved critical/high security findings.
